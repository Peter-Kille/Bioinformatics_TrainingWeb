Software

The linux OS has some really powerful utilities but these are not truly bioinformatics software. We have to ‘install’ or ‘load’ software. We do not pre-load the software as many informatic tools are not singular piece of software but are complex installs with dependencies i.e. requiring different versions of python or java and these elements can conflict causing software to malfunction.

Most managed HPC systems (not your own personal computer) manage this by using modules. These are preconfigured installers for your software packages. Cardiff systems use a module system - if this in the case you can see the software that is available try the following command:

module avail

this will bring up a software list that looks like the following:

.......

Any of these software packages can be load by simply using the module load command, e.g.

module load fastqc/0.11.9

but remember to unload the module when you are finished so it does not conflict with the next piece of software or if you want to globally remove all module the module purge command to deinstall all loadde modules.

module unload fastqc/0.11.9
or
module purge

Data

The data for your course is available to you on a pre-mapped drive location, you can see list the folder it contains using the following simple command:

ls ~/classdata/Bioinformatics

#this should yield the follow list of folders

REFS  Session1  Session2  Session3  Session4  Session5  Session6

You cans ee these folders but not write to them. Before using them each Session folder can be copied to your mydata directory. For example if you want to copy Session1 type


cp -r ~/classdata/Bioinformatics/Session1 ~/mydata/Bioinformatics/Session1

Note the inclusion of the ‘-r’ flag to allow you to copy the folder ‘recusively’ this will copy all the folders and files inside.
